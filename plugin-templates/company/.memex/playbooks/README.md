# Playbooks

SOPs and runbooks. One page per repeatable process. Suggested structure:

- **When to run this** — the trigger
- **Steps** — numbered, copy-pasteable
- **Failure modes** — what can go wrong, how to recover
- **Owner** — who maintains this playbook
